# PRD: Candidate Comments & Collaboration System

## Product

SeekHelpers BackOffice

## Module

Candidate Comments & Discussion Drawer

## Goal

Provide a centralized discussion and collaboration system around a candidate where recruiters, agencies, sales teams, managers, and internal staff can communicate, share updates, upload files, and coordinate actions without leaving the candidate list screen.

The experience should feel similar to Slack, Linear, Intercom, and Discord Threads while maintaining the SeekHelpers design language.

---

# Core User Journey

## Candidate List

Users see a list of candidates.

Each candidate row contains:

* Candidate Avatar
* Candidate Name
* Candidate ID
* Candidate Status
* Agency
* Last Activity
* Comments Button

Example:

Comments (14)

---

Clicking Comments opens a right-side drawer.

# Success Criteria

Users must be able to:

* Open candidate comments
* Browse agencies
* Open discussions
* Create comments
* Mention teammates
* Upload images
* Upload documents
* React with emojis
* Create threaded replies
* Navigate thread view
* Search discussions
* Receive notifications
* Manage large conversations efficiently

The final experience should feel like a modern enterprise collaboration tool built specifically for candidate management and recruitment workflows.
